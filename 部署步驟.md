# 修仙·問道 — GitHub Pages 部署步驟

這個資料夾（`github-pages`）裡的內容就是要上傳的全部檔案：

```
index.html      ← 遊戲本體（就是 xiuxian.html 改名）
images/         ← 所有圖檔（30 個）
```

把這兩個（`index.html` 與整個 `images` 資料夾）上傳到 GitHub repo 的「根目錄」即可。

---

## 為什麼要用 GitHub Pages？

你之前用本機檔案（`file://`）開遊戲，Edge 對 `file://` 的 localStorage 存檔不保證保留，所以關掉瀏覽器就重來。改成 GitHub Pages 後是 `https://` 網址，localStorage 會穩定留在瀏覽器，**關掉重開存檔都還在**。

---

## 部署步驟（你已有 GitHub 帳號）

### 1. 建立新 repository
1. 登入 GitHub → 右上角「＋」→ **New repository**
2. Repository name：例如 `xiuxian`（小寫、不要空格）
3. 選 **Public**（Pages 免費版需要 Public）
4. 不用勾 README，直接 **Create repository**

### 2. 上傳檔案
最簡單的網頁拖拉方式：
1. 進到剛建好的 repo 空白頁，點 **uploading an existing file**（或 Add file → Upload files）
2. 把這個資料夾裡的 **`index.html`** 和 **`images` 資料夾** 一起拖進上傳區
   - ⚠️ 要連 `images` 整個資料夾一起拖，圖才會出現
3. 下方 **Commit changes**

> 若拖資料夾不行，可改用 GitHub Desktop 或 git 指令把整個資料夾推上去。

### 3. 開啟 Pages
1. repo 上方 **Settings** → 左側 **Pages**
2. **Source** 選 **Deploy from a branch**
3. **Branch** 選 `main`、資料夾選 **`/ (root)`** → **Save**
4. 等 1～2 分鐘，頁面上方會出現你的網址：

```
https://<你的GitHub帳號>.github.io/xiuxian/
```

用這個網址玩，存檔就會穩定保留。手機 Safari／Chrome 也是用同一個網址。

---

## 存檔保險（已內建）

遊戲洞府畫面右上角新增了 **💾 存檔** 按鈕：
- **匯出存檔**：把目前進度下載成 `.json` 檔保管
- **匯入存檔**：用該檔覆蓋現有進度（換裝置、或不小心清掉瀏覽器資料時可救回）

建議偶爾匯出備份一份，最保險。

---

## 之後要更新遊戲版本？

每次我給你新版 `xiuxian.html`，你只要：
1. 到 repo 點 `index.html` → 右上鉛筆 ✏️ → 貼上新內容 → Commit
   （或重新 Upload files 覆蓋）
2. 等 1～2 分鐘 Pages 自動重新發布

圖檔沒變的話不用重傳，只更新 `index.html` 即可。

---

## 小提醒
- localStorage 是「每個網址各自獨立」：`file://`、itch、GitHub Pages 三邊的存檔不互通，要搬進度就用 💾 匯出／匯入。
- 換手機或清瀏覽器資料前，記得先匯出存檔。
